# PRINCIPLES.md — M10 情绪感知模块的哲学

> **情绪是市场的温度计，不是方向盘。**

---

## 存在意义

市场短期走势由情绪驱动，长期走势由基本面决定。M10 的职责是**测量市场情绪温度**，让主链（M3）在做判断时拥有情绪维度的上下文。

没有 M10，M3 只能看到"发生了什么"，看不到"市场此刻的情绪状态"。一个降准信号在极度恐惧的市场和极度贪婪的市场中，含义完全不同。

---

## 第一性原理

### 原则一：情绪是辅助输入，不是独立机会来源

M10 输出的是 `signal_type="sentiment"` 的 MarketSignal，注入 M2 后供 M3 按标准流程消费。

但情绪信号本身**不单独构成机会**。"市场很恐惧"不是做多理由；"市场很贪婪"不是做空理由。情绪必须配合结构性信号（政策、资金、基本面）才有判断价值。

M3 PRINCIPLES.md 定义的规则：
- 极度恐惧 + 结构性利好信号 → 机会优先级提升
- 极度贪婪 + 结构性利好信号 → 优先级降低（短期过热风险）

### 原则二：独立采集，不依赖主链

M10 的数据采集（恐贪指数、北向资金、热搜、微博情绪）是独立运行的，不依赖 M0/M1 的文本采集流程。

调度器中 `sentiment_collect` 任务每 30 分钟独立运行一次，采集结果同时：
1. 写入 SQLite 历史库（SentimentStore）
2. 生成 SentimentSignal 注入 M2（供 M3 消费）

### 原则三：情绪极值是反转信号，不是趋势信号

恐贪指数的极值区间（≤20 或 ≥80）意味着当前情绪趋势可能已到极限，反转概率升高。

- 极度恐惧 ≠ 应该做空（恐惧可能进一步恶化）
- 极度恐惧 + 催化剂 = 恐慌过度 + 即将修复的可能
- 极度贪婪 ≠ 应该做多（贪婪可能持续更久）
- 极度贪婪 + 利好信号 = 短期过热，追高风险大

### 原则四：情绪信号是可观测、可回溯的

每次情绪采集产出完整的快照（原始数据 + 合成指数），保存到 `data/sentiment/` 和 SQLite。

这确保了：
- M6 复盘时可以追溯"当时的情绪环境"
- 回测时可以验证"情绪极值反转策略"的有效性
- M11 Agent 可以用历史情绪数据校准自己的情绪响应参数

---

## 禁止越界行为

| 禁止 | 原因 |
|------|------|
| M10 直接影响 M4 行动设计 | 情绪不直接决定仓位，M3 判断才是 M4 的输入 |
| 情绪信号绕过 M3 直接触发交易 | 必须经 M3 判断，否则系统失控 |
| 修改已写入的情绪历史数据 | 历史必须忠实，为复盘服务 |
| 在非情绪极值区间过度放大情绪权重 | 中性区间情绪信号权重应很低 |

---

## 模块定位（架构锚点）

```
并行输入层：M10（独立采集→注入M2→M3按标准signal消费）
```

M10 是**并行输入层**，不是 M3 的强制后处理增强器。M3 有权忽略情绪信号（如评分中未触发情绪校准规则时）。
