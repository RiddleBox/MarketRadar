# M3 Judgment Engine — 第一性原理文档

> **信号是原材料，机会是成品。两者之间的距离，是判断力。**

---

## 存在意义

### 信号 ≠ 机会

M1 输出的是信号——一个个孤立的、已发生的市场事实变化。

但信号本身不能告诉你应该做什么。一个降准信号，在不同的市场背景下，可能意味着：
- 经济恶化、政策被迫救市（反而看空）
- 流动性宽松、推动估值修复（看多）
- 政策已被市场预期，实际影响有限（中性）

**M3 存在，是为了把孤立的信号编织成一个有意义的"叙事框架"，判断这些信号的组合是否构成一个值得采取行动的市场机会。**

M3 是系统中唯一被允许做"判断"的模块。M1 不判断，M3 必须判断——但判断必须建立在信号证据之上，不能凭空推论。

### 2026-04-18 架构矫正：M3 是推理引擎

**核心认知修正**：M3 不是"反应型判断"，而是"推理型判断"。

**反应型判断（错误）**：
```
央行降准公告（已发生） → M3判断：利好，建议买入
问题：所有人都看到了，没有alpha
```

**推理型判断（正确）**：
```
线索1: 央行行长讲话"适时降准"
线索2: 财政部提前发债
线索3: 经济数据连续3个月低于预期
线索4: 政治局会议"加大逆周期调节"

→ M3推理：降准概率80%，时间窗口2周内
→ M3判断：提前布局，等待靴子落地
→ 降准公告前已持仓，公告后卖出获利
```

**M3 的完整能力**：
1. **信号关联分析**：识别哪些信号逻辑相关，可能共同构成机会
2. **因果链推理**：基于M2的因果图谱，推断"线索A+B+C → 事件D概率80%"
3. **历史案例检索**：从M2查询相似案例，作为判断依据
4. **叙事框架构建**：回答五个核心问题（见下文）
5. **胜率赔率评估**：不对称性分析，风险收益比

## 第一性原理

### 什么构成"真正的市场机会"？

一个真正的市场机会，需要能够回答以下五个核心问题：

**问题一：为什么价格会变化？**
- 必须有具体的催化剂（来自信号）
- 必须有清晰的传导机制（A → B → 价格变化的因果链）
- 不接受"大家觉得会涨"或"历史上这个时候会涨"

**问题二：为什么是现在？**
- 机会有时间窗口，不是永久有效的
- 必须说明为什么现在（而不是三个月前或三个月后）是特别的时机
- 时间窗口必须有具体边界

**问题三：如果错了，为什么会错？**
- 必须识别关键假设
- 假设被打破时，机会是否仍然成立？
- 反向证据是什么？（有反向证据不代表不是机会，但必须正视）

**问题四：胜率和赔率大致是多少？**
- 不要求精确计算，但要有定性判断
- 潜在收益和最大可接受损失之间的关系是否合理？
- 不对称性：好的机会通常具有"有限损失 + 较大潜在收益"的不对称性

**问题五：下一步验证什么？**
- 机会不是终点，是研究的起点
- 必须给出需要进一步验证的具体问题
- 验证步骤应该是可操作的

## M3 的推理流程（2026-04-18 新增）

### 完整判断流程

```python
def judge(signals, historical_signals) -> List[OpportunityObject]:
    """
    完整判断流程：
    
    1. 信号关联分析（Step A）
       - 识别哪些信号逻辑相关
       - 分组为潜在场景
    
    2. 因果链推理（新增）
       - 从M2查询匹配的因果模式
       - 推断：当前信号组合 → 未来事件概率
       - 输出：InferredEvent列表（事件、概率、时间窗口）
    
    3. 历史案例检索（新增）
       - 从M2查询相似历史案例
       - 分析：类似信号组合在历史上如何演化
       - 输出：CaseRecord列表（信号序列、演化、结果）
    
    4. 机会判断（Step B，增强）
       - 基于：当前信号 + 推理的未来事件 + 历史案例
       - 回答五个核心问题
       - 输出：OpportunityObject或None
    """
```

### 因果链推理（核心新增能力）

**输入**：
- 当前信号列表（来自M1）
- 历史信号列表（来自M2）

**处理**：
1. 从M2查询匹配的因果模式（`query_causal_patterns()`）
2. 用LLM评估当前信号与历史模式的相似度
3. 计算未来事件的发生概率和时间窗口

**输出**：
```python
InferredEvent(
    event_description="央行降准",
    probability=0.75,
    time_window="2周内",
    reasoning="三个前置信号指向宽松政策，历史上类似组合后2周内降准概率80%",
    supporting_pattern_ids=["pattern_001", "pattern_003"],
    supporting_cases=["case_2023_03", "case_2024_02"]
)
```

### 历史案例检索（核心新增能力）

**输入**：
- 当前信号列表

**处理**：
1. 从M2查询相似历史案例（`query_similar_cases()`）
2. 提取案例的关键特征：
   - 信号序列
   - 演化过程
   - 最终结果
   - 经验教训

**输出**：
```python
CaseRecord(
    case_id="case_2023_03",
    date_range=(2023-03-10, 2023-03-24),
    signal_sequence=[
        Signal("央行行长讲话提到降准", 2023-03-10),
        Signal("财政部提前发债", 2023-03-12),
        Signal("经济数据低于预期", 2023-03-15),
    ],
    evolution="市场从3-15开始预期降准，3-20出现降准传闻",
    outcome={
        "event_occurred": True,
        "event_date": "2023-03-24",
        "market_reaction": "+3.5%",
        "time_to_event": 14
    },
    lessons="提前2周布局，降准公告当天卖出，获利3.5%"
)
```

## 推理 vs 反应：关键区别

| 维度 | 反应型判断（旧） | 推理型判断（新） |
|------|----------------|----------------|
| 输入 | 已发生的信号 | 线索组合 |
| 处理 | 评估信号影响 | 推断未来事件概率 |
| 输出 | "降准是利好" | "降准概率80%，2周内，提前布局" |
| Alpha来源 | 无（所有人都看到） | 有（先于市场共识） |
| 时间窗口 | 事后反应 | 事前预判 |
| 依据 | 当前信号 | 因果图谱+历史案例 |

**关键洞察**：
- 反应型判断：等待"央行降准公告"这个明确信号，然后判断
- 推理型判断：看到"央行行长讲话+财政发债+经济数据差"这3个线索，推断降准概率，提前布局

## 市场特异性

### A股机会判断特殊规则

**规则一：政策催化剂效力评估**
- A股政策敏感性极高，单一政策信号可能构成短期机会
- 但要区分"首次政策"和"重复喊话"——第五次提到同一政策的边际效力远低于第一次
- 检查历史上类似政策的实际落地效果

**规则二：流动性驱动 vs 基本面驱动**
- A股短期机会往往是流动性驱动（北向资金、公募发行）
- 流动性驱动的机会时间窗口短，见好就收
- 基本面驱动的机会持续时间长，可以更有耐心

**规则三：散户情绪放大效应**
- 当 MarketSentinel 的情绪信号显示市场处于情绪极值（极度恐惧或极度贪婪）时
- 极度恐惧 + 结构性利好信号 = 机会优先级提升
- 极度贪婪 + 结构性利好信号 = 机会优先级降低（短期过热）

**规则四：板块轮动逻辑**
- A股存在显著的板块轮动现象
- 单一行业长期持续上涨后，资金往往流向滞涨板块
- 识别"尚未反应"的受益板块有时比识别"直接受益"板块更有价值

### 港股机会判断特殊规则

**规则一：双重驱动要求**
- 港股理想机会需要两个市场的驱动力同时存在：
  - A股/中国政策端：正面催化剂
  - 全球流动性端：美联储政策不构成重大逆风
- 单一驱动的港股机会可靠性明显低于双重驱动

**规则二：估值折价修复逻辑**
- 港股相对A股长期折价，折价扩大/收窄本身是一类机会类型
- 南向资金持续流入 + 港股估值相对低位 = 可能的折价修复机会

**规则三：流动性分层**
- 不同流动性级别的港股机会完全不同：
  - 蓝筹（恒生权重股）：机会可以落实到具体仓位
  - 中盘股：机会存在但执行有摩擦
  - 小市值港股：即使判断正确也难以无损执行，不建议生成仓位建议

## "不构成机会"的合法出口

M3 输出空列表 `[]` 不是失败，是诚实的判断。

以下情况必须输出空列表，而非强行输出：

**情况一：信号不够聚焦**
- 信号指向不同甚至相反的方向（宏观偏负面 + 行业偏正面 + 政策偏中性），无法形成合力
- 应该等待更多信号形成方向一致性

**情况二：时间窗口已关闭或无法判断**
- 如果无法合理确定一个时间窗口，说明机会可能还不成熟
- "随时都可能发生"等于"不知道什么时候发生"，不是机会

**情况三：关键假设过于脆弱**
- 机会成立的关键假设本身就不可靠
- 例如：整个论点建立在"某某政策一定会在某时间点出台"上，但这本身是预测

**情况四：信号质量太低**
- 所有相关信号的 confidence_score 都低于 5
- 低质量信号的组合不会产生高质量机会

**情况五：已经充分定价**
- 所有相关信号的 timeliness_score 都低于 4
- 市场已经反应，没有剩余的定价空余

**情况六：推理证据不足（新增）**
- 因果图谱中没有匹配的模式
- 历史案例中没有相似情况
- 无法推断未来事件的概率和时间窗口
- 此时应输出空列表，等待更多信号积累

## 输出承诺：OpportunityObject 字段语义

| 字段 | 语义承诺 | 不允许 |
|------|---------|--------|
| `opportunity_thesis` | 完整阐述机会的逻辑链，必须回答五个核心问题 | 模糊的"市场可能上涨" |
| `why_now` | 解释时间紧迫性，为什么是现在 | 留空或复制 thesis |
| `related_signals` | 只列出真正支撑该机会的信号ID | 把所有相关信号都堆进来 |
| `inferred_events` | 推理的未来事件（新增） | 不填写或留空 |
| `supporting_cases` | 支撑的历史案例ID（新增） | 不填写或留空 |
| `key_assumptions` | 列出"如果这个假设错了，整个机会就不成立"的假设 | 列举显而易见的背景知识 |
| `counter_evidence` | 列出真实存在的反向证据 | 为了乐观而忽视反向证据 |
| `uncertainty_map` | 按重要程度排列不确定性来源 | 随意列举一些"通用风险" |
| `priority_level` | 基于时间窗口和确定性的客观评级 | 主观上感觉"好像很重要"就给 urgent |
| `risk_reward_profile` | 定性描述潜在盈利和可接受损失的关系 | 只说"潜在收益大"，不提风险 |
