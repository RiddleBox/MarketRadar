# MarketRadar 定时任务配置

## 方案 A：使用 crontab（Linux/macOS）

### 1. 编辑 crontab
```bash
crontab -e
```

### 2. 添加以下配置
```bash
# MarketRadar 每日自动化流程
# 注意：需要替换 /path/to/MarketRadar 为实际路径

# 盘前流程（周一到周五 09:00）
0 9 * * 1-5 cd /path/to/MarketRadar && /usr/bin/python3 run_daily_pipeline.py --mode premarket >> logs/daily_pipeline.log 2>&1

# 盘中流程（周一到周五 10:00, 14:00）
0 10,14 * * 1-5 cd /path/to/MarketRadar && /usr/bin/python3 run_daily_pipeline.py --mode intraday >> logs/daily_pipeline.log 2>&1

# 盘后流程（周一到周五 15:30）
30 15 * * 1-5 cd /path/to/MarketRadar && /usr/bin/python3 run_daily_pipeline.py --mode postmarket >> logs/daily_pipeline.log 2>&1
```

### 3. 验证配置
```bash
# 查看当前 crontab
crontab -l

# 测试手动运行
cd /path/to/MarketRadar
python3 run_daily_pipeline.py --mode premarket
```

## 方案 B：使用 systemd timer（Linux）

### 1. 创建服务文件
```bash
sudo nano /etc/systemd/system/marketradar-premarket.service
```

内容：
```ini
[Unit]
Description=MarketRadar Pre-Market Pipeline
After=network.target

[Service]
Type=oneshot
User=your_username
WorkingDirectory=/path/to/MarketRadar
ExecStart=/usr/bin/python3 /path/to/MarketRadar/run_daily_pipeline.py --mode premarket
StandardOutput=append:/path/to/MarketRadar/logs/daily_pipeline.log
StandardError=append:/path/to/MarketRadar/logs/daily_pipeline.log

[Install]
WantedBy=multi-user.target
```

### 2. 创建 timer 文件
```bash
sudo nano /etc/systemd/system/marketradar-premarket.timer
```

内容：
```ini
[Unit]
Description=MarketRadar Pre-Market Timer
Requires=marketradar-premarket.service

[Timer]
OnCalendar=Mon-Fri 09:00:00
Persistent=true

[Install]
WantedBy=timers.target
```

### 3. 启用并启动
```bash
sudo systemctl daemon-reload
sudo systemctl enable marketradar-premarket.timer
sudo systemctl start marketradar-premarket.timer

# 查看状态
sudo systemctl status marketradar-premarket.timer
sudo systemctl list-timers
```

### 4. 类似地创建 intraday 和 postmarket 的 service/timer

## 方案 C：使用 Windows 任务计划程序

### 1. 打开任务计划程序
```
Win + R → taskschd.msc
```

### 2. 创建基本任务
- 名称：MarketRadar Pre-Market
- 触发器：每天 09:00，仅周一到周五
- 操作：启动程序
  - 程序：`python.exe`
  - 参数：`run_daily_pipeline.py --mode premarket`
  - 起始于：`D:\AIProjects\MarketRadar`

### 3. 高级设置
- 勾选"如果任务失败，重新启动"
- 勾选"如果任务运行时间超过，停止任务"（设置为 30 分钟）
- 勾选"如果请求后任务还在运行，强行停止"

### 4. 类似地创建 intraday 和 postmarket 任务

## 方案 D：使用 Python APScheduler（跨平台）

### 1. 安装依赖
```bash
pip install apscheduler
```

### 2. 创建调度脚本
```python
# scheduler_daemon.py
from apscheduler.schedulers.blocking import BlockingScheduler
from apscheduler.triggers.cron import CronTrigger
import subprocess
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

scheduler = BlockingScheduler()

def run_pipeline(mode):
    logger.info(f"Running {mode} pipeline...")
    try:
        subprocess.run(
            ["python", "run_daily_pipeline.py", "--mode", mode],
            check=True,
            capture_output=True,
            text=True,
        )
        logger.info(f"{mode} pipeline completed")
    except subprocess.CalledProcessError as e:
        logger.error(f"{mode} pipeline failed: {e}")

# 盘前（周一到周五 09:00）
scheduler.add_job(
    lambda: run_pipeline("premarket"),
    CronTrigger(day_of_week="mon-fri", hour=9, minute=0),
    id="premarket",
)

# 盘中（周一到周五 10:00, 14:00）
scheduler.add_job(
    lambda: run_pipeline("intraday"),
    CronTrigger(day_of_week="mon-fri", hour="10,14", minute=0),
    id="intraday",
)

# 盘后（周一到周五 15:30）
scheduler.add_job(
    lambda: run_pipeline("postmarket"),
    CronTrigger(day_of_week="mon-fri", hour=15, minute=30),
    id="postmarket",
)

if __name__ == "__main__":
    logger.info("MarketRadar Scheduler started")
    try:
        scheduler.start()
    except KeyboardInterrupt:
        logger.info("Scheduler stopped")
```

### 3. 运行调度器
```bash
# 前台运行（测试）
python scheduler_daemon.py

# 后台运行（生产）
nohup python scheduler_daemon.py > logs/scheduler.log 2>&1 &
```

## 推荐方案

### Windows 用户
- **推荐**：方案 C（Windows 任务计划程序）或方案 D（APScheduler）
- 优点：图形界面，易于配置
- 缺点：需要保持电脑开机

### Linux/macOS 用户
- **推荐**：方案 A（crontab）
- 优点：简单可靠，系统原生支持
- 缺点：需要手动编辑配置文件

### 服务器部署
- **推荐**：方案 B（systemd timer）
- 优点：更好的日志管理和服务控制
- 缺点：配置稍复杂

## 日志管理

### 创建日志目录
```bash
mkdir -p logs
```

### 日志轮转（logrotate）
```bash
# /etc/logrotate.d/marketradar
/path/to/MarketRadar/logs/daily_pipeline.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
}
```

### 查看日志
```bash
# 实时查看
tail -f logs/daily_pipeline.log

# 查看最近100行
tail -n 100 logs/daily_pipeline.log

# 搜索错误
grep -i error logs/daily_pipeline.log
```

## 故障排查

### 问题 1：定时任务未执行
- 检查 cron 服务是否运行：`systemctl status cron`
- 检查日志：`grep CRON /var/log/syslog`
- 验证路径和权限：`ls -la /path/to/MarketRadar`

### 问题 2：Python 环境问题
- 使用绝对路径：`/usr/bin/python3` 或 `/path/to/venv/bin/python`
- 激活虚拟环境：`source /path/to/venv/bin/activate && python ...`

### 问题 3：权限问题
- 确保脚本可执行：`chmod +x run_daily_pipeline.py`
- 确保日志目录可写：`chmod 755 logs/`

### 问题 4：依赖缺失
- 在 crontab 中设置环境变量：
  ```bash
  PATH=/usr/local/bin:/usr/bin:/bin
  PYTHONPATH=/path/to/MarketRadar
  ```
