# M1.5 vs M3: 隐性信号推理应该放在哪里？

> **问题**: 为什么要新增M1.5模块？直接在M3做隐性推理不行吗？
> **时间**: 2026-04-24 16:15

---

## 🤔 核心问题

### 方案A: 新增M1.5
```
M1(显性信号) → M1.5(隐性信号) → M2(存储) → M3(机会判断)
```

### 方案B: 不新增M1.5
```
M1(显性信号) → M2(存储) → M3(隐性推理 + 机会判断)
```

## 🎯 方案A的优势

### 1. 符合单一职责原则
- M1: 提取事实
- M1.5: 推理信号
- M3: 判断机会

### 2. 符合开闭原则
- 新增产业链知识 → 只需更新M1.5
- 优化判断逻辑 → 只需更新M3
- 互不影响

### 3. 符合依赖倒置原则
```python
# M3依赖抽象（信号），不依赖具体（推理逻辑）
class OpportunityJudge:
    def judge(self, signals: List[Signal]) -> Opportunity:
        # 不关心信号是显性还是隐性
        # 不关心信号是怎么来的
        return self._judge(signals)
```

### 4. 符合接口隔离原则
- M6复盘：只需要M1.5（推理），不需要M3（判断）
- 用户输入：只需要M1.5（推理），不需要M3（判断）
- 回测：只需要M1.5（推理），不需要M3（判断）

## 🤔 关键判断标准

### 如果隐性推理是**简单逻辑**
```python
# 示例：简单的关键词映射
def infer_implicit(signal):
    if "降准" in signal.content:
        return [Signal(type="流动性宽松")]
    if "加息" in signal.content:
        return [Signal(type="流动性收紧")]
```

**结论**: 不需要M1.5，直接在M3做即可。

---

### 如果隐性推理是**复杂逻辑**
```python
# 示例：多阶推理
def infer_implicit(signal):
    # 1. 查询产业链图谱（100+行业，1000+关系）
    related = industry_graph.query(signal)
    
    # 2. 查询因果模式（50+模式）
    patterns = causal_graph.query(signal.type)
    
    # 3. LLM推理（需要prompt工程）
    implicit_signals = llm.infer(signal, related, patterns)
    
    # 4. 置信度评估
    for s in implicit_signals:
        s.confidence = self._evaluate_confidence(s)
    
    return implicit_signals
```

**结论**: 需要M1.5，因为：
- 逻辑复杂，需要独立测试
- 依赖知识库，需要独立管理
- 需要被复用（M6、回测、用户输入）

## 📋 替代方案：M1.5作为M3的子模块

如果你担心架构过于复杂，可以考虑：

```python
# M1.5作为M3的子模块，但暴露独立接口
class OpportunityJudge:
    def __init__(self):
        self.implicit_inferencer = ImplicitSignalInferencer()  # M1.5
    
    def judge(self, signals: List[Signal]) -> Opportunity:
        # 1. 推理隐性信号
        implicit_signals = self.implicit_inferencer.infer(signals)
        
        # 2. 判断机会
        return self._judge_opportunity(signals + implicit_signals)
    
    # 暴露M1.5接口，供其他模块复用
    def infer_implicit_signals(self, signals: List[Signal]) -> List[Signal]:
        return self.implicit_inferencer.infer(signals)
```

**优点**:
- 架构上M1.5是M3的一部分（简单）
- 接口上M1.5可以独立调用（灵活）

**缺点**:
- M1.5的生命周期绑定M3
- 不如完全独立的M1.5清晰
