## 二、距离实用工具的差距评估

### 当前定位

项目已从"研究原型"升级为"交易研究助手"。具备完整的 M1→M6 生产判断链、M7 回测、M8 知识库、M9 模拟盘、M10 情绪感知、M11 Agent 模拟（验证链）、工作流、确认机制、审计日志。

### 实用工具标准

一个"实用工具"应满足：
1. **可用性**：用户能日常使用，不需要改代码
2. **可靠性**：核心流程稳定，不频繁崩溃
3. **可解释性**：每个判断/行动可追溯
4. **可配置性**：参数可通过配置文件调整
5. **可部署性**：能在目标环境稳定运行

### 差距分析

#### 1. 数据源可用性（中优先级）

| 差距 | 说明 | 影响 | 工作量 |
|------|------|------|--------|
| TuShare token 未配置 | 需要用户注册+充值积分 | 分钟级行情不可用，降级到 AKShare | 用户操作 |
| AKShare 无鉴权 | 免费但有限流 | 高频采集可能被封 IP | 中 |
| M10 情绪采集依赖外部 API | 百度热搜/微博可能变更接口 | 情绪数据可能中断 | 中 |

**缓解**：CompositeFeed 已实现 fallback 链（tushare→akshare→csv），但需要用户配置至少一个可用数据源。

#### 2. LLM 可用性（中优先级）

| 差距 | 说明 | 影响 | 工作量 |
|------|------|------|--------|
| M3 Step B JSON 解析偶发失败 | DeepSeek 下已大幅改善但未根除 | 偶尔丢失机会判断 | 低 |
| 工蜂 AI 依赖内网 | 需要 IOA 认证 | 换机器/外网不可用 | 用户操作 |
| 无 LLM 成本监控 | 每次调用消耗 token | 长期使用成本不可控 | 低 |

**缓解**：已有 fallback 链（工蜂→DeepSeek→OpenAI-compatible），JSON 解析有 repair 机制。

#### 3. Dashboard 可用性（低优先级）

| 差距 | 说明 | 影响 | 工作量 |
|------|------|------|--------|
| 无自动刷新 | 需要手动点击刷新 | 盘中数据延迟 | 低 |
| 无止损/止盈临近提醒 | 持仓接近关键价位时无提示 | 可能错过最佳执行时机 | 低 |
| 错误处理不够友好 | `load_*()` 函数静默返回空列表 | 用户不知道数据加载失败 | 低 |

#### 4. 部署与运维（中优先级）

| 差距 | 说明 | 影响 | 工作量 |
|------|------|------|--------|
| 无 Docker 化 | 部署依赖手动安装依赖 | 换环境成本高 | 中 |
| 无进程管理 | 调度器崩溃后无自动重启 | 需要人工干预 | 低 |
| 无日志轮转 | 日志文件可能无限增长 | 磁盘占用不可控 | 低 |
| 无健康检查端点 | 无法监控服务状态 | 故障发现延迟 | 低 |

#### 5. M11 校准质量（高优先级）

| 差距 | 说明 | 影响 | 工作量 |
|------|------|------|--------|
| 方向命中率仅 50% | 低于 70% 阈值 | M11 不准入主链 | 高（需 LLM 模式校准） |
| 极值召回率 0% | 无法识别历史极值事件 | 极端行情下无预警 | 中 |
| 情绪历史估算粗糙 | 从价格数据代理，非真实 M10 采集 | 校准精度受限 | 中 |

#### 6. 组合级风控完善度（低优先级）

| 差距 | 说明 | 影响 | 工作量 |
|------|------|------|--------|
| 相关系数矩阵需要手动提供 | 无自动计算历史相关性 | 高相关去重依赖用户输入 | 低 |
| 主题映射需要手动配置 | 无自动分类标的到主题 | 主题暴露检查不完整 | 低 |

### 差距优先级排序

```
高优先级：
  → M11 校准质量改善（LLM 模式 + 真实情绪历史）

中优先级：
  → TuShare token 配置引导
  → LLM 成本监控
  → Docker 化部署
  → 进程管理（systemd/supervisor 配置）

低优先级：
  → Dashboard 自动刷新
  → 止损/止盈临近提醒
  → 错误处理增强
  → 日志轮转
  → 健康检查端点
  → 相关系数自动计算
  → 主题自动分类
```

## 三、下一步建议

### 立即可做（用户侧）

1. **配置 TuShare token**：注册 TuShare Pro，获取 token 写入 `config/tushare_config.yaml`
2. **配置 LLM provider**：确保至少一个 LLM 可用（工蜂/DeepSeek/OpenAI-compatible）
3. **运行一次端到端测试**：`python pipeline/run_pipeline.py --input test_news.txt`

### 短期改进（1-2 周）

1. **M11 LLM 模式校准**：使用 LLM 替代规则模式重新跑 60 事件，预期方向命中率提升至 65-75%
2. **Docker 化**：编写 Dockerfile + docker-compose.yml
3. **Dashboard 自动刷新**：添加 `streamlit-autorefresh`
4. **LLM 成本监控**：记录每次调用的 token 消耗和费用

### 中期改进（1-2 月）

1. **M3 历史上下文增强**：M2 相似信号检索，区分旧闻与新闻
2. **M4→M7 参数共享验证**：确保回测使用与模拟盘相同的参数
3. **进程管理**：systemd 服务配置 + 自动重启
4. **日志轮转**：logrotate 配置
