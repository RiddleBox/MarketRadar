# MarketRadar 整体规划 v1

## 1. 项目定位

MarketRadar 的目标不是做一个“会总结新闻的 LLM demo”，而是构建一个面向二级市场研究与执行辅助的模块化系统：

- 从非结构化市场信息中提取信号
- 将信号聚合为结构化机会对象
- 基于机会对象生成行动计划
- 通过回测与模拟盘验证策略有效性
- 最终演进为一个可持续辅助真实交易决策的实用性工具

当前项目已经完成统一 LLM 接入，并在 DeepSeek 下打通 `M1 -> M3 -> M4` 主链路，证明核心架构可行。

---

## 2. 总体演进阶段

### 阶段 A：研究原型阶段（已基本完成）
目标：让系统具备“事件 -> 信号 -> 机会 -> 动作”的最小闭环。

已达成：
- 统一 LLM 配置与调用入口
- 工蜂AI接入 `LLMClient`
- DeepSeek 作为可用稳定 provider 验证通过
- `M1 -> M3 -> M4` 主链路跑通
- M3 关键 schema 容错修复完成

### 阶段 B：验证增强阶段（下一阶段重点）
目标：让系统输出从“像样”升级为“可解释、可验证、可复盘”。

重点：
- M3 机会判断评分化
- 参数系统配置化
- M4 行动设计参数化
- 回测系统升级为 OpportunityObject 驱动
- 模拟盘与机会对象绑定

### 阶段 C：准实用工具阶段
目标：形成“研究 + 决策辅助 + 交易前验证 + 模拟执行”的稳定工作流。

重点：
- 情绪讨论 agent 扩展
- 组合层风险约束
- 持仓监控与失效条件跟踪
- 策略回测与模拟盘联动
- 复盘归因闭环

### 阶段 D：准交易系统阶段
目标：在严格风控与长期验证基础上，成为真实交易的高质量辅助系统。

注意：
- 在长期回测、模拟盘、风控体系未成熟前，不应让 LLM 直接控制真实交易执行。

---

## 3. 模块级规划

### M1 信号解码
定位：市场事件的结构化入口。

后续方向：
- 提高对政策、宏观、资金流、业绩、情绪类文本的识别准确率
- 引入 signal 去重与聚类
- 引入来源质量评分与时效衰减
- 增加对长文本/多源摘要的整合

### M2 信号存储
定位：统一信号资产库。

后续方向：
- 增加跨批次检索
- 增加历史相似事件索引
- 增加主题聚类、时间序列聚合
- 为 M3 提供更强上下文支持

### M3 机会判断
定位：系统的核心判断中枢。

后续方向：
- 建立显式机会评分框架
- 引入 invalidation 条件与 kill switch
- 引入市场确认、预期差、可交易性等维度
- 支持多信号聚合形成“主题机会”

### M4 行动设计
定位：把机会对象变成可执行的行动方案。

后续方向：
- 按品类区分 ETF / 股票 / 期货 的动作模板
- 让止损 / 止盈 / 仓位建议参数化
- 支持分阶段建仓、动态调整、复核条件
- 与回测系统共享执行参数

### M5 持仓管理
定位：持仓状态机与风控执行层。

后续方向：
- 管理持仓生命周期
- 监控止损/止盈/有效期/失效条件
- 形成持仓事件日志
- 与模拟盘/实盘接口解耦

### M6 复盘归因
定位：把每次机会、执行、盈亏都沉淀为系统经验。

后续方向：
- 归因是“判断错”还是“执行错”
- 归因是“参数错”还是“市场 regime 变化”
- 回流改进 M3 / M4 / 参数系统

### 情绪讨论 Agent
定位：辅助判断叙事热度、分歧程度、拥挤风险。

后续方向：
- 作为 M3 的辅助修正项，不直接替代主判断
- 输出 heat / crowding / narrative_consensus / opposing_views
- 与市场事件、资金流、价格表现结合

### 回测系统
定位：验证机会逻辑是否有持续统计优势。

后续方向：
- 从策略模板回测升级到 OpportunityObject 驱动回测
- 支持事件回测、参数敏感性分析、机会类型分桶评估
- 输出收益/回撤/胜率/盈亏比分布

### 模拟盘系统
定位：把研究建议放进近真实执行环境验证。

后续方向：
- 机会对象与模拟仓位绑定
- 引入滑点/手续费/流动性约束
- 自动止损/止盈/到期处理
- 自动复盘记录

## 4. 核心设计原则

### 原则 1：LLM 负责理解与生成，规则负责约束与验证
- 不把交易系统建立在“模型自然会输出对的格式”之上
- LLM 输出必须经过 schema + 归一化 + 风控规则约束

### 原则 2：机会对象是全系统核心中枢
- 回测、模拟盘、行动设计、复盘都要围绕 `OpportunityObject` 对齐
- 避免不同模块各说各话

### 原则 3：参数配置必须统一且外置
- 风控、执行、机会判断阈值不应散落在代码中
- 必须形成统一配置系统

### 原则 4：优先做“可验证”而不是“更聪明”
- 先让机会判断变得可解释、可评分、可回测
- 再考虑引入更复杂的情绪图谱或多 agent 体系

### 原则 5：不直接追求自动交易，先追求稳定决策辅助
- 先把研究、验证、模拟盘闭环做好
- 再考虑接近实盘的事情

---

## 5. 近期主线目标

未来一到两轮迭代的主线目标是：

1. 把 DeepSeek 固化为当前稳定联调 provider
2. 把 M3 从“单次判断”升级为“评分化机会引擎”
3. 把 M4 从“建议生成”升级为“参数化行动设计器”
4. 把回测和模拟盘统一绑定到 OpportunityObject
5. 把系统从原型推进到“验证增强阶段”

---

## 6. 最终愿景

最终目标不是单纯“发现机会”，而是让系统能够：

- 快速发现市场中重要的事件与结构性变化
- 判断这些变化是否形成可交易机会
- 输出符合风险约束的行动方案
- 在回测与模拟盘中被持续验证
- 通过复盘不断提升判断与执行质量

也就是把 MarketRadar 逐步打造为：

**一个面向真实交易研究工作流的、可解释、可验证、可迭代的市场机会操作系统。**
