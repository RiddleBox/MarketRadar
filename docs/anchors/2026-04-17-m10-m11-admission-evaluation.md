> 评估依据：架构锚点 — "验证链有效 ≠ 应进入主链"

## 二、M10 准入评估

### 证据

| 维度 | 评估 | 详情 |
|------|------|------|
| 数据质量 | ✅ 合格 | 恐贪指数/北向资金/涨跌比/微博情绪四维数据源，有 fallback |
| Schema 兼容性 | ✅ 合格 | SentimentSignalData → MarketSignal 转换完整，通过 Pydantic 校验 |
| 注入 M2 可行性 | ✅ 合格 | `run_and_inject()` 已修复，scheduler sentiment_collect 任务正常注册 |
| M3 消费能力 | ✅ 合格 | M3 已实现 `_extract_sentiment_context()` + `_calibrate_priority()` 情绪校准规则 |
| 独立性 | ✅ 合格 | 独立采集不依赖主链，失败不阻断 M1→M3 流程 |
| 失败影响 | ✅ 可控 | 采集失败仅 loss 情绪上下文，M3 仍可基于其他信号判断 |

### 角色定义

M10 作为**并行输入层**（架构锚点定义）：

1. `signal_type="sentiment"` 的 MarketSignal 经 M2 供 M3 消费
2. M3 情绪校准规则仅在极值区间（FG≤20 或 FG≥80）生效
3. M10 不直接触发 M4 行动设计
4. 中性区间（FG 40-60）情绪信号权重很低

### 约束

- M10 采集失败时 M3 不得阻塞等待
- 情绪信号不作为独立机会来源（M3 PRINCIPLES 明确）
- 情绪信号存储与普通信号无区别（M2 统一处理）

## 三、M11 准入评估

### 证据

| 维度 | 评估 | 详情 |
|------|------|------|
| 方向命中率 | ❌ 不合格 | 50%（阈值 ≥ 70%） |
| 概率校准误差 | ❌ 不合格 | 0.424（阈值 < 0.15） |
| 极值召回率 | ❌ 不合格 | 0%（阈值 ≥ 60%） |
| 综合得分 | ❌ 不合格 | 29.5（阈值 ≥ 55） |
| 事件积累 | ✅ 合格 | 60 个历史事件（目标 ≥ 50） |
| 持久化 | ✅ 合格 | CalibrationStore SQLite 持久化已实现 |

### 校准数据（60 事件，规则模式，2026-04-17）

```
Total events: 60
Direction accuracy: 50.00%
Prob calibration err: 0.424
Extreme recall: 0.00%
Composite score: 29.5
Pass threshold: False
```

### 失败原因分析

1. **规则模式太粗**：5 个 Agent 的规则逻辑是简化启发式，无法捕捉市场复杂动态
2. **情绪估算偏差**：历史情绪上下文从价格数据估算，而非真实 M10 采集，误差大
3. **极值识别缺失**：规则模式对极值事件的 intensity 评分普遍偏低（<7.0），导致 extreme_recall=0%
4. **方向判断过于机械**：简单跟随价格方向，未考虑反转场景

### 不准入的理由

按架构锚点：**"验证链有效 ≠ 应进入主链"**。M11 当前方向命中率仅 50%（等同随机），作为验证链都不够格，更不应升格为主链节点。

### 改善路径（后续迭代）

| 改善项 | 预期效果 | 优先级 | 状态 |
|--------|---------|--------|------|
| LLM 模式校准 | 提升方向命中率至 65-75% | 高 | Iteration 10 已完成：37%（2%阈值）/ 预计65%+（3%阈值+优化） |
| 修正 Agent 规则权重 | 改善 extreme_recall | 中 | 待做 |
| M10 真实情绪历史替代价格估算 | 提升概率校准精度 | 中 | 待做 |
| Agent 间交互拓扑优化 | 可能提升方向判断 | 低 | 待做 |
| 添加历史上下文（N日后事件） | 改善 BEARISH 检测 | 高 | 待做 |

### Iteration 10 LLM 校准结果更新（2026-04-18）

LLM 模式（DeepSeek）校准已完成，60 事件：

| 指标 | Rule-based (2%) | LLM (2%) | Delta |
|------|----------------|----------|-------|
| Direction Accuracy | 21.67% | 36.67% | +15.00% |
| Composite Score | 11.6 | 22.4 | +10.8 |
| BULLISH hit rate | 31% | 47% | +16% |
| BEARISH hit rate | 6% | 28% | +22% |
| NEUTRAL hit rate | 20% | 20% | 0% |

**结论**: LLM 模式显著优于规则模式（+15pp），但仍远低于 70% 准入阈值。核心瓶颈是：
1. 系统性看多偏差（政策信号过强）
2. 震荡行情识别能力不足
3. 缺乏历史上下文（如"大涨后第3天"的反转逻辑）

**M11 维持不准入主链**。下一步需优化 agent 权重和 prompt，并添加历史上下文信息。

## 四、综合决策

1. **M10 准入**：有合格的数据质量、Schema 兼容性、独立性和可控的失败影响。作为并行输入层入主链是安全的。
2. **M11 不准入**：核心校准指标全面不达标。需先在验证链内改善至 pass_threshold=True 后重新评估。
3. **下一次评估时机**：当 M11 在 LLM 模式下校准通过阈值时。
