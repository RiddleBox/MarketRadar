## Phase 2 任务分解

### 任务 1: 实现盘中流程 (优先级: 🔴 高)

**目标**: 定期更新持仓价格,检查止损/止盈触发

**文件**: `run_daily_pipeline.py` - `run_intraday()`

**实现步骤**:
1. 从 M9 PaperTrader 获取当前持仓
2. 调用价格源 (AKShare/YFinance) 更新价格
3. 检查止损/止盈触发条件
4. 触发时推送告警 (控制台 + 可选邮件)
5. 人工确认后执行平仓

**验收标准**:
- [ ] 能获取所有开仓持仓
- [ ] 能更新持仓价格 (3-5分钟延迟可接受)
- [ ] 能检测止损/止盈触发
- [ ] 触发时有明确告警
- [ ] 支持人工确认后平仓

**预计工时**: 4-6 小时

### 任务 4: 配置定时任务 (优先级: 🟡 中)

**目标**: 实现真正的自动化 (每天自动运行)

**文件**: 系统 crontab / systemd

**实现步骤**:
1. 编写 crontab 配置
2. 测试定时任务触发
3. 添加日志轮转 (避免日志文件过大)
4. 添加失败告警 (邮件/钉钉/飞书)

**Crontab 配置**:
```bash
# 盘前 (周一到周五 09:00)
0 9 * * 1-5 cd /mnt/d/AIProjects/MarketRadar && /usr/bin/python3 run_daily_pipeline.py --mode premarket >> logs/cron_premarket.log 2>&1

# 盘中 (周一到周五 10:00, 14:00)
0 10,14 * * 1-5 cd /mnt/d/AIProjects/MarketRadar && /usr/bin/python3 run_daily_pipeline.py --mode intraday >> logs/cron_intraday.log 2>&1

# 盘后 (周一到周五 15:30)
30 15 * * 1-5 cd /mnt/d/AIProjects/MarketRadar && /usr/bin/python3 run_daily_pipeline.py --mode postmarket >> logs/cron_postmarket.log 2>&1
```

**验收标准**:
- [ ] crontab 配置正确
- [ ] 定时任务能自动触发

## 风险与应对

### 风险 1: 实时数据源不稳定
**影响**: 盘中价格更新失败
**应对**: 
- 使用 AKShare (3-5分钟延迟可接受)
- 添加数据源降级逻辑
- 缓存最近价格作为备份

### 风险 2: 定时任务执行失败
**影响**: 自动化中断
**应对**:
- 添加失败重试逻辑
- 配置告警通知
- 定期检查日志

### 风险 3: 因果图谱质量不高
**影响**: 推理引擎准确率下降
**应对**:
- 先从高质量模式开始 (货币政策已验证)
- 每个新模式都要回测验证
- 定期review和优化
