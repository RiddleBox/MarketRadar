## 实用性差距（从"研究原型"到"日常工具"）

### 🔴 关键缺口 1：实时数据源不稳定

**现状**：
- ✅ AKShare：3-5分钟延迟，免费，已验证
- ✅ YFinance：15分钟延迟，免费，已验证
- ❌ iTick/AllTick：实时数据，API Key认证失败

**影响**：
- 无法做盘中实时决策（只能盘后复盘）
- 止损/止盈触发延迟（3-5分钟）
- 无法捕捉快速行情变化

**解决方案**：
1. **短期**：使用AKShare（3-5分钟延迟）+ 降低交易频率（只做日内/隔夜级别）
2. **中期**：修复iTick/AllTick API Key（需要官网激活）
3. **长期**：接入券商API（实时Level-2行情）

**优先级**：🟡 中等（短期可用AKShare绕过）

---

### 🔴 关键缺口 2：M0→M3 自动化流程未打通

**现状**：
- ✅ M0 CLI可手动采集
- ✅ M1→M6 单元测试通过
- ❌ 没有"一键运行"脚本（从采集到判断）
- ❌ M7调度器未配置定时任务

**影响**：
- 每天需要手动运行多个脚本
- 无法自动监控新信号
- 错过时效性强的机会

**解决方案**：
1. **创建端到端脚本**：`run_daily_pipeline.py`
   ```python
   # 盘前：采集 → 解码 → 判断 → 生成行动计划
   # 盘中：更新价格 → 检查止损止盈
   # 盘后：复盘 → 归因 → 更新知识库
   ```

2. **配置定时任务**：
   - 盘前（09:00）：运行M0→M3，生成今日机会列表
   - 盘中（10:00/14:00）：更新持仓，检查触发条件
   - 盘后（15:30）：运行M6复盘，更新M8知识库

3. **人工确认点**：
   - M3输出机会 → 人工审核 → M4生成行动计划
   - M4行动计划 → 人工确认 → M5开仓
   - M5触发止损/止盈 → 人工确认 → 平仓

**优先级**：🔴 高（这是"日常可用"的核心）

---

### 🔴 关键缺口 3：M3推理引擎的因果图谱太少

**现状**：
- ✅ 初始化10个货币政策因果模式
- ❌ 缺少行业/个股/技术面因果模式
- ❌ 缺少历史案例库（CaseRecord表为空）

**影响**：
- 只能识别货币政策相关机会（降准/降息）
- 无法识别行业轮动、个股事件驱动、技术面突破等机会
- 推理引擎覆盖面窄

**解决方案**：
1. **扩充因果图谱**（目标：50+模式）：
   - 货币政策（10个，已完成）
   - 行业政策（10个）：新能源补贴、地产政策、医药集采等
   - 个股事件（10个）：业绩预增、重组并购、股东增持等
   - 技术面（10个）：突破关键位、成交量放大、MACD金叉等
   - 资金面（10个）：北向资金流入、融资余额变化、大宗交易等

2. **构建历史案例库**（目标：100+案例）：
   - 从2020-2024历史数据中提取成功/失败案例
   - 每个案例包含：信号序列 → 演化过程 → 结果 → 教训
   - 用于M3推理引擎的案例检索

3. **持续学习机制**：
   - M6复盘后，将成功案例写入CaseRecord
   - 定期review失败案例，更新因果图谱

**优先级**：🟡 中等（当前10个模式可覆盖部分场景）

---

### 🟡 次要缺口 4：Dashboard缺少实时监控

**现状**：
- ✅ Dashboard有6个tab（信号/机会/持仓/复盘/知识/审计）
- ❌ 需要手动刷新，无实时推送
- ❌ 无告警机制（止损触发、新机会出现）

**影响**：
- 无法第一时间知道止损触发
- 错过时效性强的机会

**解决方案**：
1. **WebSocket实时推送**：
   - M5持仓状态变化 → 推送到Dashboard
   - M3新机会生成 → 推送到Dashboard

2. **告警机制**：
   - 止损/止盈触发 → 桌面通知 + 邮件
   - 高优先级机会 → 桌面通知

**优先级**：🟢 低（短期可手动刷新）

---

### 🟡 次要缺口 5：M0数据源覆盖不全

**现状**：
- ✅ 4个数据源：新闻/公告/研报/市场数据
- ❌ 缺少社交媒体（雪球/微博/Reddit）
- ❌ 缺少政策文件（发改委/央行/证监会官网）
- ❌ 缺少财报数据（业绩快报/年报/季报）

**影响**：
- 错过部分信号源
- 推理引擎输入不完整

**解决方案**：
1. **扩充M0 providers**：
   - 社交媒体：雪球API、微博API
   - 政策文件：爬虫 + RSS订阅
   - 财报数据：AKShare财报接口

2. **优先级排序**：
   - 政策文件（高）：对M3推理引擎最有价值
   - 财报数据（中）：对个股判断有价值
   - 社交媒体（低）：噪音大，需要过滤

**优先级**：🟢 低（当前4个数据源可覆盖主要场景）
