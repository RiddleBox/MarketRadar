# MarketRadar 快速开始指南

> 从零到每日自动生成机会列表，只需 10 分钟

---

## 前置条件

### 1. 环境要求
- Python 3.9+
- Git
- 10GB 磁盘空间

### 2. API Key（可选）
- OpenAI API Key（用于 LLM 推理）
- 或使用本地模型（Ollama/LM Studio）

## 快速开始（10 分钟）

### Step 1: 克隆项目（1 分钟）
```bash
git clone https://github.com/RiddleBox/MarketRadar.git
cd MarketRadar
```

### Step 2: 安装依赖（2 分钟）
```bash
# 安装核心依赖
pip install -r requirements.txt

# 或使用虚拟环境（推荐）
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### Step 3: 配置环境变量（1 分钟）
```bash
# 复制配置模板
cp .env.example .env

# 编辑 .env，填入 API Key
nano .env
```

`.env` 内容：
```bash
# LLM 配置
OPENAI_API_KEY=sk-your-api-key-here
OPENAI_BASE_URL=https://api.openai.com/v1  # 可选：使用代理或本地模型

# 数据源配置（可选）
ITICK_API_KEY=your-itick-key  # 实时行情（可选）
ALLTICK_API_KEY=your-alltick-key  # 实时行情（可选）
```

### Step 4: 初始化数据库（1 分钟）
```bash
# 创建必要目录
mkdir -p data/incoming logs

# 初始化因果图谱（10个货币政策模式）
python scripts/init_causal_graph.py
```

输出：
```
✓ 初始化 10 个因果模式
✓ 数据库就绪
```

### Step 5: 测试运行（5 分钟）

## 常见问题

### Q1: OpenAI API 调用失败
**问题**：`openai.error.AuthenticationError`

**解决**：
1. 检查 `.env` 中的 `OPENAI_API_KEY` 是否正确
2. 检查 API Key 是否有余额
3. 或使用本地模型（见下文）

### Q2: 使用本地模型（不需要 OpenAI API）
**方案 A：Ollama**
```bash
# 1. 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 2. 下载模型
ollama pull qwen2.5:14b

# 3. 修改 .env
OPENAI_BASE_URL=http://localhost:11434/v1
OPENAI_API_KEY=ollama  # 任意值
```

**方案 B：LM Studio**
1. 下载 LM Studio：https://lmstudio.ai/
2. 加载模型（推荐 Qwen2.5-14B）
3. 启动本地服务器（默认端口 1234）
4. 修改 `.env`：
   ```bash
   OPENAI_BASE_URL=http://localhost:1234/v1
   OPENAI_API_KEY=lmstudio
   ```

### Q3: M0 采集无数据
**问题**：`无新数据，跳过后续步骤`

**原因**：
- RSS 源可能暂时无更新
- 或所有数据已被去重

**解决**：
```bash
# 强制重新采集（跳过去重）
python m0_collector/cli.py --source rss --force-reimport

# 或手动添加测试数据
echo "央行宣布降准0.5个百分点" > data/incoming/test.txt
python run_daily_pipeline.py --mode premarket
```

### Q4: M3 未识别到机会
**问题**：`识别机会: 0 个`

**原因**：
- 信号不足（需要多个信号组合）
- 或当前信号不匹配因果图谱

**解决**：
1. 积累更多信号（运行几天后效果更好）
2. 扩充因果图谱（见下文）

### Q5: 如何扩充因果图谱？
```bash
# 查看当前因果模式
python -c "
from m2_storage.signal_store import SignalStore
store = SignalStore()
patterns = store.query_causal_patterns()
for p in patterns:
    print(f'{p.pattern_id}: {p.consequent_event} (概率{p.probability})')
"

# 手动添加新模式（示例）
python -c "
from m2_storage.signal_store import SignalStore
from core.schemas import CausalPattern
from datetime import datetime

store = SignalStore()
pattern = CausalPattern(
    pattern_id='pattern_custom_001',
    precursor_signals=['新能源补贴政策', '新能源车销量增长'],
    consequent_event='新能源板块上涨',
    probability=0.70,
    avg_lead_time_days=7,
    std_lead_time_days=2.0,
    supporting_cases=[],
    confidence=0.60,
    last_updated=datetime.now(),
    created_at=datetime.now(),
    notes='自定义模式：新能源政策驱动',
)
store.save_causal_pattern(pattern)
print('✓ 添加成功')
"
```

## 核心特性

### ✅ 推理型判断（不是反应型）
- **反应型**：看到降准公告才判断（所有人都看到了，无 alpha）
- **推理型**：识别前置信号组合，提前 14 天预测降准概率 80%

### ✅ 因果图谱 + 历史案例
- 10 个初始因果模式（货币政策）
- 支持扩展到 50+ 模式（行业/个股/技术/资金）
- 历史案例检索（从 2020-2024 数据中学习）

### ✅ 端到端自动化
- 盘前：采集 → 解码 → 判断 → 生成机会列表
- 盘中：更新价格 → 检查止损止盈
- 盘后：复盘归因 → 更新知识库

### ✅ 人工确认点
- M3 机会审核（人工决定是否执行）
- M4 行动计划确认（人工调整仓位）
- M5 开仓确认（人工最终确认）
