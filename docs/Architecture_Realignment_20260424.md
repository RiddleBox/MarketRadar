# MarketRadar 架构重新对齐报告

> **时间**: 2026-04-24 16:00
> **触发**: 用户澄清核心目标 - 从非财经信息中提取财经信号
> **目的**: 基于第一性原理重新审视架构，确认是否需要新增模块

---

## 🎯 核心目标澄清

### 之前的理解（偏差）
```
财经新闻 → 直接信号 → 投资机会
```
这是**一阶信息**，所有人都能看到，**没有信息不对称优势**。

### 正确的目标
```
非财经信息 → 隐含逻辑链 → 产业机会 → 投资标的
```
这是**二阶/三阶推理**，从大家忽略的信息碎片中提前发现机会。

---

## 🔍 当前架构审视

### M0 收集器 - **需要扩展**

**当前状态**:
- ✅ 采集财经新闻（AKShare、Finnhub）
- ❌ 缺少非财经信息源

**第一性原理**:
> M0的职责是**忠实采集原始信息**，不做解读。

**问题诊断**:
- 数据源过于狭窄，只有财经媒体
- 缺少政策、技术、社会、国际等维度

**改进方向**:
```python
# 当前
M0 = 财经新闻采集器

# 应该是
M0 = 多维信息采集器
  ├─ 财经维度: 东方财富、财联社、证券时报
  ├─ 政策维度: 新华社、发改委、工信部
  ├─ 技术维度: 36氪、虎嗅、学术期刊
  ├─ 社会维度: 微博热搜、小红书、抖音
  ├─ 国际维度: 路透社、彭博、外交部
  └─ 产业维度: 海关数据、招聘信息、专利数据
```

---

### M1 信号解码 - **需要升级**

**当前状态**:
- ✅ 提取直接信号（政策/业绩/资金）
- ❌ 缺少隐含信号提取能力

**第一性原理**:
> M1的职责是**提取事实，不做预测**。

**问题诊断**:
- 只能识别显性信号（"央行降准"）
- 无法识别隐性信号（"中沙合作" → 光伏订单预期）

**改进方向**:
```python
# 当前
M1: 提取显性信号
  输入: "央行宣布降准50bp"
  输出: Signal(type="货币政策", direction="宽松")

# 应该是
M1: 提取显性+隐性信号
  输入: "中国-沙特签署新能源合作备忘录"
  输出: [
    Signal(type="外交合作", direction="正面"),  # 显性
    Signal(type="订单预期", affected=["光伏"], confidence=0.7),  # 隐性
    Signal(type="产业链受益", affected=["硅料","逆变器"], confidence=0.5)  # 隐性
  ]
```

**关键**: 隐性信号仍然是**事实陈述**（"存在订单预期"），不是预测（"股价会涨"）。

---

### M2 信号存储 - **需要增强**

**当前状态**:
- ✅ 存储信号
- ✅ 因果图谱（10个模式）
- ✅ 历史案例（4个案例）
- ❌ 缺少产业链知识库

**第一性原理**:
> M2的职责是**存储信号 + 提供推理支撑**。

**问题诊断**:
- 因果图谱只有货币政策维度
- 缺少产业链上下游关系
- 缺少事件→标的映射

**改进方向**:
```python
# 当前
M2 = SignalStore + CausalGraph + CaseLibrary

# 应该是
M2 = SignalStore + CausalGraph + CaseLibrary + IndustryGraph
  ├─ SignalStore: 信号存储（已有）
  ├─ CausalGraph: 因果图谱（需扩充）
  │   ├─ 货币政策 → 行业影响（已有10个）
  │   ├─ 外交合作 → 产业机会（待补充）
  │   ├─ 技术突破 → 替代关系（待补充）
  │   └─ 消费趋势 → 业绩改善（待补充）
  ├─ CaseLibrary: 历史案例（需扩充）
  │   └─ 从4个扩充到100+
  └─ IndustryGraph: 产业链图谱（新增）
      ├─ 上下游关系: 光伏 → 硅料 → 多晶硅设备
      ├─ 替代关系: 钙钛矿 vs 硅基
      └─ 受益传导: 政策 → 行业 → 公司
```

---

### M3 机会判断 - **需要升级**

**当前状态**:
- ✅ 两阶段判断（Step A/B）
- ✅ 因果链推理
- ✅ 案例检索
- ❌ 缺少多阶推理能力

**第一性原理**:
> M3的职责是**推理型判断**，回答"这是不是机会？"

**问题诊断**:
- 只能做一阶推理（信号 → 机会）
- 无法做多阶推理（事件 → L1 → L2 → L3 → 标的）

**改进方向**:
```python
# 当前
M3: 一阶推理
  信号: "央行降准"
  推理: 货币宽松 → 成长股受益
  机会: 买入科技ETF

# 应该是
M3: 多阶推理
  信号: "中沙签署新能源合作"
  推理链:
    L1: 沙特需要光伏/储能技术（外交意图）
    L2: 国内光伏龙头可能获得订单（产业映射）
    L3: 供应链公司提前受益（传导路径）
  时间窗口: 3-6个月（备忘录 → 实际订单）
  确定性: 中等（需要后续跟踪）
  机会: 买入隆基绿能/通威股份
```

### M4/M5/M6 - **无需改动**

**当前状态**: ✅ 符合第一性原理

**理由**:
- M4: 行动设计（止损/仓位）
- M5: 持仓管理（执行）
- M6: 复盘归因（学习）

这三个模块的职责与信息来源无关，无需改动。

## 🆕 是否需要新增模块？

### 方案A: 不新增模块，增强现有模块

```
M0: 扩展数据源（财经 → 多维信息）
M1: 升级解码器（显性 → 显性+隐性）
M2: 增强知识库（+产业链图谱）
M3: 升级推理引擎（一阶 → 多阶）
```

**优点**:
- 架构简洁，不增加复杂度
- 符合现有模块的第一性原理

**缺点**:
- M1职责变重（既要提取显性，又要提取隐性）
- M3职责变重（既要判断机会，又要做多阶推理）

---

### 方案B: 新增M1.5模块（推荐）

```
M0: 采集多维信息
  ↓
M1: 提取显性信号（保持简单）
  ↓
M1.5: 推理隐性信号（新增）★
  ↓
M2: 存储所有信号 + 知识库
  ↓
M3: 机会判断（多阶推理）
```

**M1.5 - 隐性信号推理器**

**职责**:
1. 输入: M1提取的显性信号
2. 推理: 基于产业链图谱，推导隐性信号
3. 输出: 隐性信号列表

**示例**:
```python
# 输入（M1输出）
Signal(type="外交合作", content="中沙签署新能源合作")

# M1.5推理
→ 查询产业链图谱: "新能源" → ["光伏", "储能", "新能源车"]
→ 查询因果模式: "外交合作" → "订单预期"
→ 查询上下游: "光伏" → ["硅料", "逆变器", "组件"]

# 输出（M1.5输出）
[
  Signal(type="订单预期", affected=["隆基绿能","通威股份"], confidence=0.7),
  Signal(type="供应链受益", affected=["阳光电源","特变电工"], confidence=0.5)
]
```

**优点**:
- 职责清晰：M1提取事实，M1.5推理隐含
- M3专注于机会判断，不承担推理负担
- 可以独立测试和优化

**缺点**:
- 增加一个模块，架构复杂度+1

---

### 方案C: 新增M2.5模块（产业链映射器）

M0 → M1 → M2 → M2.5 → M3

### 第二步: 实现M1.5模块（下周）

**核心功能**:
```python
class ImplicitSignalInferencer:
    """隐性信号推理器"""
    
    def __init__(self, industry_graph: IndustryGraph):
        self.industry_graph = industry_graph
        self.llm = LLMClient()
    
    def infer(self, explicit_signal: Signal) -> List[Signal]:
        """从显性信号推理隐性信号"""
        # 1. 查询产业链图谱
        related_industries = self.industry_graph.query(explicit_signal)
        
        # 2. 查询因果模式
        causal_patterns = self.causal_graph.query(explicit_signal.type)
        
        # 3. LLM推理
        implicit_signals = self.llm.infer_implicit_signals(
            explicit_signal, 
            related_industries, 
            causal_patterns
        )
        
        return implicit_signals
```

**优先级**: 🔴 高（核心能力）

## ✅ 架构对齐结论

### 核心问题
**之前**: 系统只能处理一阶信息（财经新闻），没有信息不对称优势。

**现在**: 需要处理二阶/三阶信息（非财经信息 → 隐含逻辑链 → 产业机会）。

### 解决方案
1. **扩展M0**: 多维信息采集（政策/技术/社会/国际）
2. **新增M1.5**: 隐性信号推理器
3. **增强M2**: 产业链知识图谱
4. **升级M3**: 多阶因果推理

### 架构调整
```
# 之前
M0(财经新闻) → M1(显性信号) → M2 → M3(一阶推理) → M4

# 现在
M0(多维信息) → M1(显性信号) → M1.5(隐性信号) → M2(+产业链图谱) → M3(多阶推理) → M4
```

### 第一性原理验证
- ✅ M0: 忠实采集（扩展数据源，不改变职责）
- ✅ M1: 提取事实（保持简单，只做显性）
- ✅ M1.5: 推理隐含（新增，职责清晰）
- ✅ M2: 存储+支撑（增强知识库）
- ✅ M3: 推理判断（升级推理能力）
